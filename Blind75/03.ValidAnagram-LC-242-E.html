Valid Anagram — LeetCode 242 (E)

1. What is the problem?

LeetCode 242 — Valid Anagram

Given two strings s and t, return true if t is an anagram of s (same characters with same frequencies).

Example 1

  • Input: s = "anagram", t = "nagaram"
  • Output: true

Example 2

  • Input: s = "rat", t = "car"
  • Output: false

2. Easy Solutions

2.1 Sorting

Sort both strings and compare. If sorted(s) == sorted(t) then it's an anagram. Time: O(n log n). Space: O(n) or O(1) depending on the sort implementation.

Python

class Solution:
    def isAnagram(self, s: str, t: str) -> bool:
        return sorted(s) == sorted(t)

Golang

func isAnagram(s string, t string) bool {
    if len(s) != len(t) { return false }
    sBytes, tBytes := []byte(s), []byte(t)
    sort.Slice(sBytes, func(i, j int) bool { return sBytes[i] < sBytes[j] })
    sort.Slice(tBytes, func(i, j int) bool { return tBytes[i] < tBytes[j] })
    return string(sBytes) == string(tBytes)
}

2.2 Hash Map / Counter

Count occurrences of each character in both strings. Time: O(n). Space: O(n).

Python

from collections import Counter

class Solution:
    def isAnagram(self, s: str, t: str) -> bool:
        return Counter(s) == Counter(t)

Golang

func isAnagramHash(s string, t string) bool {
    if len(s) != len(t) { return false }
    count := make(map[rune]int)
    for _, c := range s { count[c]++ }
    for _, c := range t { count[c]-- }
    for _, val := range count {
        if val != 0 { return false }
    }
    return true
}

3. Optimized Solution (Frequency Array)

Idea: Because the problem specifies that the inputs contain only lowercase English letters, we can use a fixed-size array of 26 instead of a Hash Map. This improves memory usage to exactly O(1) and avoids the overhead of hashing. Time: O(n).

Python

class Solution:
    def isAnagram(self, s: str, t: str) -> bool:
        if len(s) != len(t):
            return False
            
        count = [0] * 26
        for i in range(len(s)):
            count[ord(s[i]) - ord('a')] += 1
            count[ord(t[i]) - ord('a')] -= 1
            
        for c in count:
            if c != 0:
                return False
        return True

Golang

func isAnagramArray(s string, t string) bool {
    if len(s) != len(t) {
        return false
    }
    
    var count [26]int
    for i := 0; i < len(s); i++ {
        count[s[i]-'a']++
        count[t[i]-'a']--
    }
    
    for _, c := range count {
        if c != 0 {
            return false
        }
    }
    return true
}

4. Walkthrough

s = "anagram", t = "nagaram". Lengths equal. Count for s: a=3, n=1, g=1, r=1, m=1. After t: all 0. True.

5. Edge Cases

  1. Different lengths: false.
  2. Empty both: true.
  3. Unicode: Use map instead of array if not just 'a'-'z'.
  4. Single char: same char → true.

6. Pitfalls

  1. Assuming lowercase: Problem says lowercase English; use ord(c)-'a'. For full Unicode use map.
  2. Only one loop: Need to count both strings (or one increment, one decrement).
  3. Sorting: O(n log n); count is O(n).

7. Follow-ups

  1. Group anagrams (LC 49)? Map: sorted tuple of chars (or count signature) → list of strings.
  2. Anagram with unicode? Use map[rune]int.
  3. All anagram pairs? Group by signature, then for each group with k strings, pairs = k*(k-1)/2.

8. When to Use

  • Anagram / same multiset: Count array or map; or sort.
  • Pattern: Frequency count; same idea in “ransom note”, “group anagrams”.
